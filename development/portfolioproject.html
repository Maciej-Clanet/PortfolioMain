Project Overview

I needed an all-in-one location to showcase my latest development and design projects, my CV, and contact information, so I have developed this portfolio website. I have developed this website using simple HTML, CSS, and JS and hosted it on GitHub pages.

Why no framework?

To keep it simple, While the design might look more like an application than a website, and I do admit part of me was tempted to reach for React, it was not the best choice. Using React would make the bundle size unnecessarily large, reducing load times without providing any real benefit to the website's purpose of displaying static information.

When I was making the decision, my main consideration was to make the website as light and cheap to host as possible. Since this portfolio does not need to be dynamic, even something like HTMX would have been overkill! A pure no-framework static website achieves all of my requirements perfectly.

Design

As much as I would like to take full credit for the design, this portfolio was very heavily inspired by SJ Zhang. You can find his portfolio here.


That being said, I have made a lot of changes to the design in order to better suit my needs, including adding entire new pages not present in the design I took inspiration from. The CV, contact information, and individual project pages have not been present in the original design source. The design of pretty much every single page has changed substantially; after all, what I require from my portfolio is different from what SJ Zhang requires from his.
